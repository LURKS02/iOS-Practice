# IBOutlet의 weak 참조

#### IBOutlet의 요소들이 weak로 선언되는 이유
iOS 사용자 인터페이스는 뷰 계층구조(view hierarchy)를 사용하여 구성되며, 이러한 계층 구조는 일련의 뷰(view)와 서브뷰(subview)들로 이루어져 있다.<br>
각 뷰는 자신의 서브뷰를 소유하고 있다. (Ex. 버튼은 상위 뷰에 추가되어 상위 뷰는 해당 버튼을 소유한다.)<br>
이때 상위 뷰가 UI 요소를 소유한다는 것은, 상위 뷰가 해당 UI 요소에 대한 강한(strong) 참조를 유지하고 있다는 것을 의미한다.<br>
따라서 상위 뷰가 메모리에 남아있는 한 해당 UI요소도 메모리에서 해제되지 않는다.<br>

<br>

IBOutlet을 통해 ViewController가 해당 UI에 대한 참조를 가져올 때, 이미 상위 뷰가 해당 UI요소를 소유하고 있기 때문에 약한(weak) 참조를 사용한다.<br>
이는 순환 참조를 방지하고 메모리 관리를 효과적으로 수행하기 위함이다.<br>
이렇게 해서 ViewController는 UI요소를 업데이트 하거나, UI요소의 이벤트를 처리하는 역할을 수행할 수 있다.<br>
즉, ViewController는 해당 요소에 대한 로직만을 담당하며, 실제 UI 요소의 소유권과 생명주기는 상위 뷰가 관리하는 것이다.<br>
